grpc-mcp

bridge any grpc service to the model context protocol

    ┌─────────┐       ┌─────────┐
    │  gRPC   │──────▶│   MCP   │
    │ Service │◀──────│  Server │
    └─────────┘       └─────────┘
     protobuf           tools
     reflection         prompts
install
curl -sfL https://peter-trerotola.github.io/grpc-mcp/install.sh | sh

or

go install github.com/peter-trerotola/grpc-mcp/cmd/grpc-mcp-server@latest

linux/darwin/windows · amd64/arm64

config
# grpc-mcp.yaml — plaintext local dev
server:
  name: "grpc-mcp"
  transport: "stdio"

endpoints:
  - name: "local-api"
    address: "localhost:50051"
    tls:
      enabled: false
# grpc-mcp.yaml — bearer token auth
server:
  name: "grpc-mcp"
  transport: "stdio"

endpoints:
  - name: "auth-api"
    address: "api.example.com:443"
    auth:
      type: "bearer"
      bearerToken: "${AUTH_TOKEN}"
    tls:
      enabled: true
# grpc-mcp.yaml — mutual TLS
server:
  name: "grpc-mcp"
  transport: "stdio"

endpoints:
  - name: "secure-internal"
    address: "secure.internal:443"
    auth:
      type: "mtls"
      mtls:
        certFile: "/certs/client.pem"
        keyFile: "/certs/client-key.pem"
        caFile: "/certs/ca.pem"
features

auto-discovery

uses grpc reflection to discover services, methods, and message schemas

any grpc service

connect to any grpc server — no code generation or proto files needed

flexible auth

bearer tokens, api keys, mutual tls, or plaintext for local dev

works everywhere

claude desktop, claude code, cursor, any mcp client. stdio + sse transport
